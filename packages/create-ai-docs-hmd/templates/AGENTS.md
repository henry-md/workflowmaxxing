# Agent Instructions

Use `agent-docs/` as the durable project memory for coding agents.

Canonical docs:

- `agent-docs/general-directions.md`: Project-wide workflow rules and standing instructions.
- `agent-docs/coding-conventions.md`: Required implementation rules, commands, dependency constraints, migrations, and verification checks.
- `agent-docs/ui-ux-conventions.md`: Required product, design, layout, and interaction conventions.
- `agent-docs/architecture/`: Codebase structure, major systems, data flow, and service boundaries.
- `agent-docs/bug-fixes/`: Short notes on important bugs that may regress.
- `agent-docs/regression-tests/`: Focused checks for behavior that has broken repeatedly.

Before editing code, read the docs relevant to the area you are touching. When adding notes, keep them concise and reusable.
